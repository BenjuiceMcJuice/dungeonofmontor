## Bug Fixes (11 total)
- **Ambush combat lock** — when junk pile ambush triggered combat and player won initiative, `currentTurnIndex` pointed to player but `combatPhase` was forced to `enemyWindup`. Fix: ambush now adjusts `currentTurnIndex` to first enemy in turn order
- **FEAR turn skip button flash** — `isPlayerTurn` became true before condition tick ran, showing action buttons for one frame. Fix: gated on `playerConditionTicked === true`
- **Zone door black screen** — `handleOpenZoneDoor()` didn't call `setGamePhase('doors')` after `setZone()`. Added the missing call
- **Duplicate condition sprites** — FEAR and POISON from gifts were creating duplicate condition entries. Fix: all gift conditions now route through `applyCondition`
- **Poison refresh resetting drains** — re-poisoning a target cleared accumulated `drainedStats`. Fix: re-poison preserves existing drains
- **Rings/amulets can't be equipped** — Equip button missing for new slot types. Added ring/amulet to `isEquippable` check
- **'undefined' stats on new items** — detail panel crashed on helmets/boots/rings/amulets. Fixed stat display for all new item types
- **Shields in wrong inventory tab** — shields appeared in Wear instead of Arms. Moved to Arms tab, Wear excludes offhand
- **Stats panel missing equipment bonuses** — only checked weapon/armour/shield. All 3 stat panels now scan all equipment slots
- **Stat tiles showing confusing D&D modifier** — removed modifier display, now shows gear bonus or nothing
- **Render fallthrough returns null (black screen)** — replaced `return null` at end of render chain with recovery UI ("Something went wrong" + "Return to Doors" button)

## Design Decisions
- Multiple conditions coexisting makes combat more tactical — no more "BLEED replaces POISON"
- Condition enhancer items create build synergies (Rusty Nail + dagger = bleed build)
- Tailor/Peddler split gives merchants distinct identity and makes CHA useful for equipment
- ErrorBoundary is pragmatic — iOS WebKit crashes are hard to repro without on-device error capture
- Generic starter items prevent confusion about Montor-named gear at the very start
- Deep Clean risk increase makes the risk/reward decision meaningful — real danger for real loot

## Session 5 — Playtesting Fixes & Gift Picker

### Bug Fixes
- **React.Fragment crash on iOS** — zone door chamber used `React.Fragment` which isn't available in production builds on iOS Safari. Replaced with `<>` shorthand. This was THE "black screen on Underground" bug.
- **Zone door loses room state** — switching zones via zone door used stale floor data, losing all visited/cleared/corpse state. Now saves live zone state back before switching.
- **Combat lock on reflect kill** — enemies dying from Mirror Shield reflect/counter during their own attack didn't trigger victory check. Player got stuck attacking a dead enemy. Added victory check after defensive gift effects.
- **Premium item duplicate in tailor** — same item appearing as both regular stock and CHA-gated premium. Excluded premium ID from random pool.
- **Dancing Shoes "undefined DEF"** — 5 instances of `item.defBonus` without `|| 0` guard across Game.jsx and Preparation.jsx.
- **Auto-equip helmets/boots** — tavern shop now auto-equips helmets and boots on purchase like weapons/armour.
