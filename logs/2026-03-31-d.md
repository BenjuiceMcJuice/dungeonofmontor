# 2026-03-31 (session 4) — Dungeon of Montor

## Session Summary
Massive session covering conditions system, item expansion, balance tuning, levelling, UI improvements, and extensive design work on Montor's Gifts and floor structure.

## What Was Built

### Conditions system (`src/lib/conditions.js`)
- 14 conditions across Body (6) and Mind (8) slots
- Body: BLEED (stacking DoT), POISON (stat drain), BURN (burst AoE), FROST (-AGI), NAUSEA (skip chance), SLUGGISH
- Mind: FEAR (-2 rolls + freeze at low HP), FRENZY (+STR -DEF random target), CHARM (skip), DAZE (force glancing), BORED, SAD (block items), BLIND (miss chance), BLOODLUST
- BLEED stacks up to 5x, lasts until cured — patient, inevitable
- POISON drains STR → AGI → DEF cycle each tick — weakening
- BURN does 5 burst damage + 3 AoE to adjacent enemies — explosive
- Conditions tick at turn start, can kill, can skip turns
- One per slot — new condition in same slot replaces old
- Enemy innate conditions: rats=BLEED, slugs=NAUSEA, orcs=FEAR, rocks=DAZE, wraiths=BLIND

### Conditions wired into combat (`src/lib/combat.js`)
- `tickTurnStart()` processes conditions before each combatant acts
- Weapon enchantments apply conditions on hit (governed by INT stat)
- Enemy attacks apply innate conditions based on archetype
- DAZE forces glancing blow tier, BLIND adds 50% miss chance
- BURN AoE spreads fire damage to adjacent enemies on tick
- POISON accumulated stat drain tracked and applied to combat rolls
- Flee blocked by BLOODLUST, items blocked by SAD — shown in UI

### In-run levelling
- 50 XP: +5 max HP
- 120 XP: pick +1 to STR/DEF/AGI/WIS/INT
- 250 XP: +5 max HP + stat pick
- 400 XP: stat pick
- Level-up UI on victory screen with stat choice buttons
- Stats update character AND battle state

### Balance pass
- DEF reduction: floor(DEF/2) → floor(DEF/3), minimum damage 1 → 2
- Knight HP: 35 → 45
- Rat: STR 10→8, HP 12→10, DEF 8→6
- Slug: STR 10→8, HP 20→16, DEF 6→4, weaponDie d6→d4
- Garden level 1: always 1 enemy (no swarms early)
- Garden level 2: 1-2 enemies (was 2-3)
- Boss generation fixed: uses zone-specific archetype+tier (was always Void orc)
- Bosses get 3x HP, +2 STR, +2 DEF on top of tier stats
- Enemy name fix: Slate Rat "Iron Rat" → "Sewer Rat", Iron Golem → "Steel Golem"

## Design Work (documented, not built)

### Montor's Gifts system (`docs/specs/10_Montors_Gifts.md`)
- 6 gifts aligned to 7 floors: Petal, Stone, Bile, Blood, Ember, Void Shard
- Each floor has a treasure to break: Montor's Favourite Gnome, Gravy Boat, Toilet Seat, Music Box, Best Tongs, Night Light
- Gifts are Montor's power stolen against his will — he demands sacrifice
- Activate at safe rooms: Montor chooses sacrifice (LCK influences severity), you roll d20 + WIS
- Apply to Body/Mind/Weapon/Item — each combination = specific boon
- Activated boons persist across runs (Stage 2+)
- Replacing a filled slot gives upgraded version of new boon
- Cross-slot fusion: Body + Mind boons together = bonus synergy effect

### Floor structure aligned to gifts
- Grounds (Petal) → Underground (Stone) → Underbelly (Bile) → Quarters (Blood) → Works (Ember) → Deep (Void Shard) → Montor's Domain
- Dungeon as Montor's home: garden → reception → sewers → private rooms → workshop → caves → throne

### Stats unlocked
- WIS = boon power (shrine stat)
- INT = condition application chance (enchantment stat)
- LCK = sacrifice severity (mercy stat)

### Room exploration (banked)
- Chambers should have multiple interactable objects
- Rest/merchants/chests as room entities (partially done)
- Montor's Gifts hidden, found through searching
- Full spec deferred — rooms as explorable spaces with multiple objects

## Key Decisions
- BLEED/POISON/BURN are distinct: stacking DoT vs stat drain vs burst AoE
- INT governs condition application, WIS governs boon power, LCK governs sacrifice mercy
- Gifts are stolen from Montor, not given — narrative of defiance
- Activated boons persist across runs, unactivated gifts lost on run end
- Replacing filled boon slot = upgraded version (not fusion within slot)
- Cross-slot Body+Mind fusion = automatic synergy bonus
- Room exploration (multiple objects per chamber) banked for future spec
