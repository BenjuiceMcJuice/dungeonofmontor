# 2026-03-31 — Dungeon of Montor

## Session Summary
Built the 4x4 dungeon grid system for Montor's Garden (Floor 0). Replaced the linear "Encounter 1/2" combat loop with a proper zone-based navigation system using the architecture from `08_Dungeon_Architecture.md`.

## What Happened

### New: Dungeon grid system (`src/lib/dungeon.js`)
- 4x4 grid of 16 chambers per zone, matching the spec
- Maze generation via randomised DFS — all 16 chambers always reachable
- Extra random doors added (3-5) to create loops and reduce frustration
- Garden zone template pool: Entry, Rest, 5x Combat Standard, Combat Elite, Mini Boss, Merchant, Quest NPC, Trap, Loot, Event, Hidden, Stairwell Descent
- Entry fixed at position 0 (top-left), Descent fixed at position 15 (bottom-right)
- Chamber content generated on entry (enemies, loot, traps, etc.)
- Data model includes `visited`, `cleared`, `revealed`, `breadcrumbed` flags for future map visibility features

### New: Dungeon map component (`src/components/DungeonMap.jsx`)
- Visual 4x4 grid showing chambers, doors between them, current position
- Fog of war: unvisited chambers show as "?"
- Adjacent reachable chambers highlighted as navigable
- Cleared chambers show "done", uncleared visited show "!"
- Chamber icons by type (swords for combat, diamond for loot, etc.)
- Player position marked with "YOU" badge

### New: Chamber view component (`src/components/ChamberView.jsx`)
- Handles all non-combat chamber interactions: Rest, Merchant, Loot, Trap, Quest NPC, Event, Stairwell
- Rest: heals 25% max HP
- Merchant: buy potions/items with gold
- Loot/Hidden: claim gold
- Trap: take damage
- Quest NPC: help for gold reward
- Stairwell Descent: "Descend" button = win condition

### Updated: Game.jsx — complete rewrite
- Replaced linear encounter flow with dungeon grid navigation
- Game phases: map → entering → chamber/combat → back to map
- Player HP persists across all chambers in the zone
- Gold tracking throughout the run
- Combat system entirely preserved — same dice, same sprites, same party bar
- Victory returns to map; defeat ends the run
- Stairwell Descent = zone complete = run win

### Updated: Results.jsx
- Shows "Chambers Cleared" instead of "Encounters"
- Shows gold earned on victory

### 4-tier attack system (replaces binary hit/miss)
- dice.js: new `d20Attack()` function — returns tier 1-4 based on roll + modifier vs fixed bands
  - Nat 20 = Critical Hit (5%), 11+ = Hit (45%), 5-10 = Glancing Blow (30%), 1-4 = Miss (20%)
  - STR modifier shifts effective roll, making higher tiers more likely
- combat.js: `calculateTierDamage()` — Crit = 1.5x, Hit = full, Glancing = half, Miss = 0
- CombatRoller: shows tier name (Critical Hit / Hit / Glancing Blow / Missed), damage die on glancing too
- Game.jsx: combat log messages reflect tier ("devastating blow", "cleanly", "glancing blow", "misses entirely")
- Removed TN-based hit system entirely — no more "need to roll X or higher"
- CombatRoller damage display bug fixed — stale closure was showing random damage instead of actual resolved damage (ref fix)

## Key Decisions
- Entry always top-left (index 0), Descent always bottom-right (index 15) — gives the grid a clear start/end direction
- Extra maze doors (3-5 loops) prevent dead-end frustration on mobile
- Combat system unchanged — just wrapped in navigation. No combat regressions.
- `revealed` and `breadcrumbed` flags on chambers are dormant — ready for torch/loaf mechanics later
- First-person perspective by default — no map visible. Map is a future tab, gated by items/skills

### Combat narrative and display improvements
- CombatRoller rewritten: progressive sentence builds as dice resolve
- Pre-roll text generic ("attacks"), tier-specific only after d20 resolves
- d8 die shows raw weapon roll, sentence explains STR/DEF narratively
- Damage number: colour by tier (yellow=glancing, orange=hit, crimson=crit), font size scales with damage amount
- DEF narrative: when DEF exceeds hit, says "armour absorbs most of the blow" instead of showing nonsensical numbers
- Combat log colours match tier scheme (crimson/orange/yellow/faint)
- Crit damage multiplier bumped from 1.5x to 2x — rare crits should feel devastating

### Flee mechanic
- Flee button appears on player turn alongside attack
- 4-tier AGI check: crit success (clean exit), success (messy, 5-10% HP), failure (bad, 15-25% HP + gold), crit failure (blocked, 25-35% HP + gold, combat continues)
- Flee returns player to previous chamber
- Crit failure wastes the turn — enemies get to attack
- Flee narrative appears in combat log

### Stat rebalance — D&D-friendly ranges
- Enemy base stats raised so all combat stats land in 8-18 range across tiers
- Tier multipliers narrowed: dust 0.8x, slate 0.9x, iron 1.0x, crimson 1.15x, void 1.3x
- Minimum stat floor of 8 enforced on STR/DEF (keeps modifiers >= -1)
- Dust rat now: STR 8 (+0 mod), DEF 8, d4 weapon, 12 HP — can hurt you but not hard
- Knight rebalanced: STR 14 (+2), DEF 10 (+0), AGI 10 (+0), HP 35 (was 40)
- DEF reduction now reasonable — Knight floor(10/2) = 5, not 6. Rats actually deal damage.
- Damage calculation order fixed: raw → tier multiplier → DEF. Crits punch through armour.
- Damage breakdown now shown on screen: "d8:5+2=7 -4def = 3" style
- combat.js calculateTierDamage returns full breakdown object for display
