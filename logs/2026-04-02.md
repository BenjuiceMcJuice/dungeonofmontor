## Weapon System Overhaul

### 5 Base Weapon Classes
| Class | Hand | Die | Identity |
|---|---|---|---|
| Dagger | Light | d4 | Double strike, +init, dual wield |
| Sword | Light | d6 | +accuracy, balanced |
| Spear | Light | d6 | +init (higher than dagger) |
| Mace | Heavy | d6 | DEF ignore, stagger, -AGI |
| Battle Axe | Heavy | d10 | Big damage, stagger, -AGI, -accuracy |

### 4-Tier Weapon System
1. **Base** (common) — starter shop, no frills
2. **Montor's Gifted** (uncommon/rare) — one per condition per weapon class, HAS giftPower
3. **Montor's Hardened** (rare/epic) — strong stats, NO giftPower
4. **Unique** (legendary) — rule-breaking special effects

### Shields
- New cheap base Shield (6g, +1 DEF, 10% block)
- Montor's shields with condition-on-block effects (Chopping Board/BLEED, Frying Pan/BURN, Mirror/DAZE)
- Hardened shield (epic, +5 DEF, 30% block)
- **Heavy weapons (mace, axe) cannot equip shields** — enforced in both Game.jsx and Preparation.jsx

### Stagger Mechanic
- Heavy weapons have `staggerChance` property (15-50%)
- On hit, chance to apply DAZE (guaranteed stun) to enemy
- Logged in combat: "Stagger! [enemy] is Dazed!"

### Other Changes
- Removed `speed` property from all weapons (was cosmetic only, never used mechanically)
- Removed short sword / long sword distinction — merged into single Sword class
- Starter shop updated: all 5 base weapons + base shield + leather + consumables
- Loot tables fully rewritten to distribute new weapons across all 7 zones
- Weapon detail panel now shows hand type (light/heavy) instead of speed

## Design Decisions
- Mace DEF ignore nerfed from 50% to 25% at base tier (Montor weapons keep higher values)
- Spear vs sword: init vs accuracy (different niches)
- Heavy weapon trade-off: no shield slot, compensated by stagger + raw damage/DEF ignore
- Three offhand playstyles: light+shield (defensive), heavy (offensive), dual daggers (glass cannon)
- Frost is now a "setup" condition (freeze then hit harder), Stone gift path is viable
- Daze is now the stagger payoff — proper stun worth building around

## Junk Piles (major feature)
- **New files**: `src/data/junk.json`, `src/lib/junkpiles.js`
- Junk item pools per floor (7 floors × 10 items each = 70 junk items)
- Condition hazards per floor (themed: garden=POISON/NAUSEA, deep=FROST/BLIND etc.)
- Pile descriptions per floor per size
- 3 pile sizes: small (1 search), medium (2), large (3)
- PER-based search: d20+PER → fumble/poor/decent/good/excellent
- Each layer: gold, junk item, real item chance, enemy chance, condition hazard
- Terminal placement: one per zone, hidden in medium/large piles
- Removed chest, trap, hidden room types → replaced with `empty` (junk-filled rooms)
- Search UI in Game.jsx: tap pile → PER roll → results overlay
- Junk bag: separate inventory tab, stackable items with sell values
- XP from searching: 3-15 per layer, scaled by search quality
