# 2026-04-06 — Full Effects Audit + Fixes

## What happened
Deep audit of every effect system in the game — conditions, weapon procs, throwables, relic passives, gift effects. Found and fixed multiple issues.

## Bugs fixed

### Throwable button missing for most throwable types
- **File:** `dom-app/src/pages/Game.jsx` (line ~5978)
- `hasThrowables` check only matched 3 of 13 throwable effect types (`damage_all_enemies`, `condition_all_enemies`, `damage_and_condition_all`)
- Missing: `condition_one_enemy`, `condition_multi_enemies`, `damage_multi_enemies`, `timed_bomb`, `reflect_next_attack`, `wet_all_and_heal`, `risky_throw`, `damage_and_condition_one`, `debuff_all_enemies`, `summon_ally`
- Now matches all 13 types — Throw button appears whenever you have any throwable

### Condition UI persistence after expiry
- **File:** `dom-app/src/pages/Game.jsx` (enemy windup useEffect)
- Enemy condition tick happened but `setBattle()` was delayed by timeouts — UI showed old conditions while narrative said "wore off"
- Added immediate `setBattle(tickedBattle)` after tick so condition icons update instantly

### Sepsis damage too weak
- **File:** `dom-app/src/lib/conditions.js`
- Was: 30% chance to double DoT (doubling 1-2 damage = barely noticeable)
- Now: 30% chance to double DoT + 4 flat bonus sepsis damage on top
- BLEED 3 stacks + POISON = up to 10+4 = 14 damage on a SEPSIS proc

## Basic run persistence (new feature)
- localStorage (instant) + Firestore (debounced 4s backup)
- Saves at room boundaries: door pick, combat victory, floor transition, safe room done, merchant close
- Does NOT save mid-combat — resume puts you at room entrance, enemies re-spawn
- Resume prompt on load: shows character name, floor, HP, gold, level
- Start Fresh clears the save
- Death/victory clears the save via onEndRun
- New files: `src/lib/runSave.js`, `src/hooks/useRunSave.js`, `src/components/ResumeRunPrompt.jsx`
- Modified: `App.jsx` (save check, resume flow), `Game.jsx` (restore + save triggers)

## Combat victory fix
- Immediate `setBattle(tickedBattle)` in enemy windup useEffect was re-triggering the same useEffect (battle in dep array), creating a race condition where dead enemies got re-ticked and victory never fired
- Removed that line — condition icon staleness is lesser issue than broken combat

## Single-target throwables fix
- `condition_one_enemy`, `damage_and_condition_one`, `risky_throw` effects required `selectedTarget` but throw UI didn't enforce selection
- Added `throwTarget` fallback — auto-targets selected enemy or first living enemy
- Fixed 3 code paths: conditionOneEnemy, conditionOneEnemy2, healEnemy

## Universal weapon gifts fix
- ALL weapon gifts got `appliedWeaponType` tagged at equip time
- But universal gifts (Acid Edge, Toilet Hands, Ignite, Bloodletter, etc.) need it unset to enter the universal code path
- Now only Petal (class-specific, object-keyed weapon data) gets tagged
- Array-keyed weapon data (Stone/Bile/Blood/Ember/Void) stays universal

## Acid Edge nerf
- Was: guaranteed -2 DEF per hit → now: 50% proc, -1 DEF
- Added `chance` field to gifts.json, wired into Game.jsx via `rollGiftChance`

## Napalm fix
